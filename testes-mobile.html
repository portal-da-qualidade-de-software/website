# Testes Unitários

  O Teste Unitário é uma forma de verificar o funcionamento de pequenos pedaços de código, seu principal objetivo é verificar o quão bem seu código está escrito e, caso não esteja funcionando da maneira esperada, permitir que seja corrigido.

  Dessa maneira, o teste unitário vem como uma forma mais simples de testagem de códigos, já que não é preciso removê-lo após a testagem, se tornando uma forma diferenciada e eficiente de realizar esse processo tão importante no desenvolvimento de software.

# Testes de Integração

  Teste de integração é quando os módulos (unidades de código) são testados em grupo para garantir que não haja nenhuma quebra naquilo que foi feito unitariamente e naquilo que está sendo integrado.

  O cenário ideal é que sejam feitos testes de unidades primeiro e, depois disso, seja feito o teste de integração que busca compreender se os módulos funcionarão juntos.

# Testes de Sistema

  O teste de sistema é realizado após o sistema estar pronto, sendo avaliados todos os componentes e funcionalidades. O objetivo do teste de sistema é exercitar todo o software, assegurando que todos os elementos que compõem o sistema estão de acordo com as especificações dos requisitos, incluindo todos os itens de hardware e software que compõem a regra de negócio.

# Testes de Aceitação do Usuário

  Para garantir a aceitação do sistema antes de lançar o produto você pode apresentar uma versão beta a algumas pessoas a fim de realizar últimos ajustes para quando o software for ao mercado tenha êxito real.

# Testes de Regressão

  O teste de regressão é uma técnica de teste de software que consiste na aplicação de versões mais recentes do software, para garantir que não surjam novos defeitos em componentes já comprovados. Se, ao juntar o novo componente ou suas alterações com os componentes restantes do sistema surgirem novos defeitos em componentes inalterados, então considere que o sistema foi corrigido.

# Célula de Testes

  Com a Célula de Testes você otimizar o investimento com processos implementação do processo de QA, com esse modelo construímos uma esteira de trabalho para averiguação da qualidade e posicionamos de forma analítica e gerencial a empresa com os artefatos gerados pela equipe de testes, que ajudarão a analisar os impactos e riscos operacionais existentes, ajudando a empresa nas tomadas de decisão.

# Automação de Testes

  Automação de testes é o uso de software para controlar a execução de testes de sistema através da aplicação de estratégias e ferramentas, por isso cada vez mais empresas reconhecem a importância de automatizar testes. A automação de testes está diretamente relacionada à QUALIDADE do software ou sistema que será entregue ao seu cliente ou que está sendo utilizado por uma empresa, um dos fatores decisivos para o sucesso do seu negócio. Seus objetivos são a redução do envolvimento humano em atividades manuais, de tempo demandado e de custo final.
